# Problem Statement

*Source: [x] Discovery conversation — 2026-06-16*

-----

## Problem

A Christmas tree ornament should act as a Disney MagicBand-style interactive prop: tap an RFID card/band to toggle the tree lights on and off with a satisfying light and sound effect. The hardware works but the code has grown messy through iteration and has several bugs.

## Target User

Zach — prop builder / maker. Seasonal decoration for home use. Aesthetic quality matters (it's a prop).

## Goals

- Tap RFID → ornament LEDs animate → tree lights toggle on or off
- Clean, reliable code that's easy to tweak each season
- Ornament looks good: discrete wire to floor power box, polished LED animations
- Optional: whitelist specific RFID UIDs so only authorized bands work

## Core Features

- MFRC522 RFID reader detects tap
- NeoPixel animations on two rings (outer 16px, inner 11px) for visual feedback
- Relay toggles 120V AC to tree lights (on/off toggle per scan)
- Optional: DFPlayer Mini audio cue on scan

## Constraints

- Arduino Uno (ATmega328) — limited RAM, 5V logic
- Two NeoPixel strips of different sizes (16px outer, 11px inner)
- Relay on pin 7 switches mains power — safety-critical, no glitches
- Discrete wire from ornament to floor power box must stay hidden
- Pin 10 conflict between RFID SS and DFPlayer RX if audio is enabled simultaneously

## Open Questions

- Is audio (DFPlayer) being kept or dropped?
- Should UID filtering be re-enabled to restrict to specific bands?
- What should happen on "lights on" vs "lights off" — different animations?
- Any new animations or color schemes planned?
