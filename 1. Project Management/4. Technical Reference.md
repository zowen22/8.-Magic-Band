# Technical Reference

*Last updated: 2026-06-16*

-----

## Hardware

| Component | Part | Notes |
|---|---|---|
| Microcontroller | Arduino Uno (ATmega328) | 5V logic, ~32KB flash, 2KB RAM |
| RFID Reader | MFRC522 | SPI interface |
| NeoPixels (outer) | 16-pixel ring | Pin 5, NEO_GRB |
| NeoPixels (inner) | 11-pixel ring | Pin 6, NEO_GRB |
| Relay | Single-channel 5V relay | Pin 7, switches 120V mains to tree lights |
| Audio (optional) | DFPlayer Mini | SoftwareSerial — currently disabled (pin conflict) |

## Pin Map

| Pin | Function |
|---|---|
| 5 | NeoPixel outer ring (16px) |
| 6 | NeoPixel inner ring (11px) |
| 7 | Relay control (HIGH = on) |
| 9 | MFRC522 RST |
| 10 | MFRC522 SS (SPI) ⚠️ conflicts with DFPlayer RX if audio enabled |
| 11 | SPI MOSI / DFPlayer TX |
| 12 | SPI MISO |
| 13 | SPI SCK |

## Libraries

| Library | Purpose |
|---|---|

## Known Bugs (Rev2 / FINAL)

| Bug | Location | Impact |
|---|---|---|
| `rainbow2` inner loop condition commented out (`10` instead of `i<pixels.numPixels()`) | `rainbow2()` | Infinite inner loop — hangs the sketch |
| `rainbowstill()` uses uninitialized `j` | `rainbowstill()` | Undefined behavior — random garbage color offset |
| `Wheel()` has dead code after `return` for pixelsIn | `Wheel()` | pixelsIn always gets same color as pixels (unreachable block) |
| `disneyaccess/disneyoff` address pixels 0–23 but NUMPIXELS1=16, NUMPIXELS2=11 | multiple functions | Out-of-bounds pixel writes (silent on NeoPixel library but wastes cycles) |
| Relay toggle duplicated in both branches of counter check | `loop()` | Redundant code, easy to desync |
| `counter` starts at 2 and uses parity — `TreeOn` boolean declared but unused | `loop()` | Confusing, fragile |
| SS_PIN (10) and PIN_MP3_RX (10) conflict | globals | DFPlayer can't run while RFID SPI is active |
| UID check commented out — any card toggles relay | `loop()` | Intended for now, but a security/reliability gap |

## System Flow

```
Boot → RFID idle loop
  ↓ card detected
Play disney() animation (3x) × inner ring chasing light
  ↓
Toggle relay (tree on/off)
  ↓ if turning ON
  Rainbow animation on both rings
  ↓ if turning OFF
  disneyoff() — clear all pixels
  ↓
2.5s debounce delay → back to idle
```

## Known Constraints

- ATmega328 has no hardware multiply for gamma correction — `pixels.gamma32()` is slow; avoid in tight loops
- NeoPixel `show()` disables interrupts briefly — don't call in ISR context
- Relay is wired to 120V mains — never leave relay pin floating on boot (boot sets LOW = off)
- `while (!Serial)` in FINAL version blocks startup without USB — removed in Rev2, keep it removed
